# 需求文档

## 简介
本功能为一个本地 PDF 知识库系统，面向希望学习 RAG 检索层基础知识的开发者。系统支持上传 PDF 文件、提取文本、分块处理、生成嵌入向量并存储，最终通过自然语言查询实现语义搜索，返回最相关的文本块。本系统专注于检索（Retrieval）部分，不涉及 LLM 生成回答。

此外，系统对外暴露一组只读 JSON API，供外部编排层（如自然语言 Agent）以程序方式调用既有的检索能力（语义搜索、文档列表、文档详情）。Agent 编排与 LLM 生成逻辑本身不在本规格范围内，由独立规格（typescript-agent）承担。

## 边界上下文

- **范围内**：PDF 上传、文本提取、固定大小分块、嵌入生成、向量存储、语义搜索（前 5 个结果）、三个页面（上传页 / 文档详情页 / 搜索页）、面向外部消费者的只读 JSON API（语义搜索 / 文档列表 / 文档详情）、原始 PDF 文件的持久化存储与整份文件查看链接
- **范围外**：LLM 生成回答、Agent 编排循环、聊天界面、用户认证、多用户支持、异步任务队列、API 的写操作（创建/更新/删除文档）、原文页级深链（跳转到来源具体页码——固定字符分块不记录页码，列为后续增强）
- **相邻期望**：
  - 文档处理为同步阻塞方式，上传操作触发完整 ingestion pipeline，结果返回前页面保持等待状态；系统不依赖后台任务队列
  - JSON API 为只读层，复用既有检索逻辑，不改变既有 HTML 页面（上传 / 详情 / 搜索）的行为；API 消费者负责自身的认证与编排，本系统不为 API 提供鉴权

## 需求

### 需求 1：PDF 文档上传

**目标**：作为学习者，我希望能上传 PDF 文件，以便系统处理并存储文档内容。

#### 验收标准

1. When 用户提交包含有效 PDF 文件的上传表单，the 知识库系统 shall 接收文件并启动 ingestion 处理流程
2. When ingestion 流程完成，the 知识库系统 shall 将用户重定向至该文档的详情页
3. If 用户提交非 PDF 格式的文件，then the 知识库系统 shall 拒绝处理并向用户展示错误提示
4. If 上传文件为空或大小为零，then the 知识库系统 shall 拒绝处理并向用户展示错误提示

---

### 需求 2：文本提取

**目标**：作为学习者，我希望系统能自动从 PDF 中提取文本，以便后续分块和嵌入处理。

#### 验收标准

1. When PDF 文件被成功上传，the 知识库系统 shall 从 PDF 的所有页面中提取可识别文本
2. If PDF 文件不包含可提取文本（如纯图像型 PDF），then the 知识库系统 shall 记录提取失败状态，并在文档详情页展示相应提示
3. The 知识库系统 shall 保留每段提取文本与其所属文档的关联关系

---

### 需求 3：文本分块

**目标**：作为学习者，我希望提取后的文本被切分为适当大小的块，以便理解分块对检索质量的影响。

#### 验收标准

1. When 文本提取完成，the 知识库系统 shall 将提取文本切分为固定大小的文本块
2. The 知识库系统 shall 将每个文本块与其所属文档关联存储
3. When 分块完成，the 知识库系统 shall 在文档详情页展示该文档生成的文本块总数

---

### 需求 4：嵌入生成与向量存储

**目标**：作为学习者，我希望每个文本块都能被转换为嵌入向量并持久化存储，以便支持语义搜索。

#### 验收标准

1. When 文本分块完成，the 知识库系统 shall 为每个文本块生成对应的嵌入向量
2. The 知识库系统 shall 将每个文本块的内容与其嵌入向量一同持久化存储
3. If 某个文本块的嵌入生成失败，then the 知识库系统 shall 记录失败状态，并在文档详情页展示处理失败提示

---

### 需求 5：文档详情查看

**目标**：作为学习者，我希望查看已上传文档的处理状态和统计信息，以便确认 ingestion 结果。

#### 验收标准

1. When 用户访问文档详情页，the 知识库系统 shall 展示文档文件名、上传时间、文本块数量和当前处理状态
2. While ingestion 流程进行中，the 知识库系统 shall 在详情页展示"处理中"状态
3. When ingestion 流程成功完成，the 知识库系统 shall 在详情页展示"处理完成"状态及生成的文本块数量
4. When ingestion 流程因错误终止，the 知识库系统 shall 在详情页展示"处理失败"状态及错误原因
5. When 用户访问已存储原始文件的文档详情页，the 知识库系统 shall 提供一个指向该文档原始 PDF 的查看链接

---

### 需求 6：语义搜索

**目标**：作为学习者，我希望通过自然语言问题对知识库进行语义搜索，以便找到最相关的文本内容。

#### 验收标准

1. When 用户在搜索页提交非空查询文本，the 知识库系统 shall 基于向量相似度返回最相关的前 5 个文本块
2. The 知识库系统 shall 按相似度从高到低的顺序排列搜索结果
3. If 知识库中没有任何已处理完成的文档，the 知识库系统 shall 在搜索结果区显示"暂无可搜索的文档"提示
4. If 用户提交空查询，then the 知识库系统 shall 阻止提交并提示用户输入查询内容

---

### 需求 7：搜索结果展示

**目标**：作为学习者，我希望清晰地看到搜索结果的内容和来源，以便直观理解语义检索的效果。

#### 验收标准

1. When 搜索返回结果，the 知识库系统 shall 展示每个结果块的文本内容及其所属文档名称
2. The 知识库系统 shall 最多展示 5 个搜索结果
3. When 用户提交新查询，the 知识库系统 shall 在不进行整页刷新的情况下更新搜索结果区域

---

### 需求 8：语义搜索 API（程序化访问）

**目标**：作为外部编排层（Agent），我希望以程序方式提交自然语言查询并获得结构化的搜索结果，以便将检索结果用于后续回答生成。

#### 验收标准

1. When API 消费者向语义搜索端点提交包含非空查询文本的请求，the 知识库系统 shall 返回基于向量相似度的最相关前 5 个文本块，以 JSON 格式包含每个文本块的内容、其所属文档文件名及指向该文档原始 PDF 的查看链接
2. The 知识库系统 shall 按相似度从高到低的顺序排列返回的文本块
3. The 知识库系统 shall 以 JSON 格式返回搜索响应，不包含 HTML 包装
4. If API 消费者提交的查询文本为空或缺失，then the 知识库系统 shall 返回客户端错误状态码及 JSON 格式的错误说明，且不执行搜索
5. If 知识库中没有任何已处理完成的文本块，then the 知识库系统 shall 返回空结果列表而非错误状态

---

### 需求 9：文档列表 API

**目标**：作为外部编排层（Agent），我希望以程序方式获取知识库中所有文档的清单，以便回答"知识库里有哪些资料"这类问题。

#### 验收标准

1. When API 消费者请求文档列表端点，the 知识库系统 shall 以 JSON 数组返回所有文档，每项包含文档标识符、文件名、处理状态、文本块数量及指向该文档原始 PDF 的查看链接
2. If 知识库中没有任何文档，then the 知识库系统 shall 返回空 JSON 数组而非错误状态
3. The 知识库系统 shall 以 JSON 格式返回文档列表响应，不包含 HTML 包装

---

### 需求 10：文档详情 API

**目标**：作为外部编排层（Agent），我希望以程序方式获取指定文档的详细信息，以便回答关于特定文档的问题。

#### 验收标准

1. When API 消费者以有效文档标识符请求文档详情端点，the 知识库系统 shall 以 JSON 返回该文档的文件名、上传时间、处理状态、文本块数量、错误信息（若存在）及指向该文档原始 PDF 的查看链接
2. If 请求的文档标识符在知识库中不存在，then the 知识库系统 shall 返回"未找到"状态码及 JSON 格式的错误说明
3. The 知识库系统 shall 以 JSON 格式返回文档详情响应，不包含 HTML 包装

---

### 需求 11：原始文件存储与查看

**目标**：作为学习者与外部编排层（Agent），我希望系统保留上传的原始 PDF 并提供整份文件的查看链接，以便从搜索结果或文档详情直接打开来源文件核对原文。

#### 验收标准

1. When 用户成功上传有效 PDF 文件，the 知识库系统 shall 持久化存储该原始 PDF 文件，并与对应文档记录保持关联
2. When API 消费者或用户通过查看链接请求某文档的原始文件，the 知识库系统 shall 返回该文档存储的原始 PDF 文件内容
3. If 请求的文档标识符不存在或其原始文件缺失，then the 知识库系统 shall 返回"未找到"状态
4. The 知识库系统 shall 为每个已存储原始文件的文档提供一个稳定的查看链接，供 HTML 页面与 JSON API 共同引用
