使い方
Chat UI + Knowledge Base UI + REST API
Info-Agentは2つの主要インターフェースを提供します。AI対話インターフェース（:3000）でドキュメントを自然言語で照会できます。ナレッジベースインターフェース（:8080）ではPDFのアップロード、ドキュメント管理、セマンティック検索を行えます。
AI対話 :3000 ナレッジベース :8080 PDFアップロード セマンティック検索 Django Admin

💬 AI対話インターフェース（localhost:3000）

💡
Agentは自動的にナレッジベースを検索するタイミングを判断します
「ドキュメントを検索してください」と明示的に指示しなくても、Agentは質問の内容から search_knowledge_base などのツールを呼び出す必要があるかを自動判断し、回答に出典情報を付与します。

↓ ドキュメント検索が必要な場合

→
pgvector検索
↓ 結果を統合して回答
✅ 回答 + 出典情報
↘ LLMエラー時
❌ サービス一時停止（エラーメッセージ）

使用例：

📝 一般的な照会
「Aプロジェクトの仕様書の主要なポイントは何ですか？」
「アップロードしたドキュメントのセキュリティポリシーについて説明してください」
「2つの技術資料の違いを比較してください」
📋 ドキュメント管理照会
「現在アップロードされているドキュメントは何ですか？」→ Agentが list_documents を呼び出し
「特定のドキュメントの詳細情報は？」→ Agentが get_document_detail を呼び出し
🔄
マルチターン会話のサポート： システムは自動的に sessionId を割り当て、同じブラウザタブでの会話履歴を保持します。追加質問や多段階の会話が可能です。タブを閉じると会話記録は消去されます（メモリ保存）。

📤 PDFアップロード（localhost:8080）

📄 PDFファイルを選択
→
⬆️ アップロード（HTTP POST）
→
⏳ ステータス：pending
↓ 同期処理
📑 pymupdf テキスト抽出
→
✂️ テキスト分割（Chunking）
↓
🔢 MiniLM ベクトル埋め込み（384次元）
→
💾 pgvectorに保存
→
✅ ステータス：complete
↘ 画像型PDF時
❌ ステータス：failed（テキスト抽出不可）
⚠️
画像型PDFは非対応
スキャン式PDF（テキストが画像として保存）はテキスト抽出できないため、処理後のステータスが「失敗」になります。選択可能なテキストが含まれているPDFをご確認ください。

アップロード時にドキュメントのカテゴリを選択できます。カテゴリによってベクトル化のChunkサイズ戦略が変わります：

resume（職歴書）
400文字 / 100重複
policy（規程）
500文字 / 120重複
manual（操作マニュアル）
800文字 / 200重複
technical（技術資料）
1000文字 / 200重複
report（レポート）
800文字 / 150重複
other（その他）
700文字 / 150重複
💡
なぜChunkサイズをカテゴリ別に変えるのか？
職歴書の各項目は短くて独立しているため、小さいChunkでベクトルをより精確にします。技術資料は長い説明文があるため、大きいChunkで文脈を保持し、意味の断片化を防ぎます。

🔍 セマンティック検索（ナレッジベース Web UI）

🧠
セマンティック検索 vs キーワード検索
本システムはベクトルのコサイン類似度検索を使用しているため、意味的に近い語彙も理解できます。例えば「人事規定」で検索すると「勤務規程」を含むドキュメントも見つかります。正確なキーワードは不要です。

http://localhost:8080 の検索ボックスにクエリ（自然言語またはキーワード）を入力すると、最も関連性の高いテキスト断片（デフォルト top-5）が返されます。

検索結果のフォーマット
• ドキュメント名 — 元PDFのファイル名
• 関連テキスト断片 — 最も関連性の高いChunkの内容
• コサイン距離 — 小さいほど関連性が高い（0 = 完全一致）

📊 ドキュメント処理ステータス

⏳ pending
処理待ち（アップロード直後の初期状態）
⚙️ processing
テキスト抽出とベクトル化を実行中
✅ complete
処理完了、AI検索対応済み（Chunk数表示）
❌ failed
処理失敗（エラーメッセージ表示、主に画像型PDF）

🌐 REST API（AgentまたはcURLで直接呼び出し可能）

メソッドパス説明
GET /api/search/?q=クエリ ベクトル類似度検索、top-5の関連Chunkを返す
GET /api/search/?q=クエリ&category=technical カテゴリ絞り込み検索（特定のドキュメント種別のみ）
GET /api/documents/ 全ドキュメント一覧取得（ステータス、Chunk数を含む）
GET /api/documents/<id>/ 特定ドキュメントの詳細情報取得
Shell 
# セマンティック検索の例
curl "http://localhost:8080/api/search/?q=プロジェクト仕様"

# カテゴリ絞り込み検索
curl "http://localhost:8080/api/search/?q=技術アーキテクチャ&category=technical"

# ドキュメント一覧
curl "http://localhost:8080/api/documents/"
