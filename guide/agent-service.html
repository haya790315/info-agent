❓ 設計上の決定

💡
なぜプロンプト注入（直接RAG検索）ではなくFunction Callingを使うのか？
従来のRAGは毎回の対話前に検索してコンテキストを注入しますが、これは文書不要な質問でもトークンを消費します。Function CallingによりLLMが「検索が必要か」「どのキーワードで検索するか」を自律的に判断でき、より効率的かつ正確な検索意図が得られます。
💡
なぜ最大ToolRoundsを5ではなく8に設定するのか？
多言語クエリのシナリオ（例：日本語文書に中国語で質問）では、Agentが検索キーワードを複数回変更する必要があります（3回検索×往復 = 6+ラウンド）。8に設定することで多言語再検索シナリオに対応しつつ、無限ループを防ぎます。
✅
なぜ出典はモデルに依存せず、バックエンドで強制付与するのか？
LLMは出典を省略したり、フォーマットが不一致になることがあります。出典付与ロジックをbuildSourceBlock()（agent.ts:73）で決定論的に処理することで、検索結果の上位3つの重複のないドキュメント名が必ず回答の末尾に表示されます。

// agent/src/agent.ts — AgentLoopのコア（簡略版）
for (let round = 0; round < maxToolRounds; round++) {
  const turn = await llm.chat(working, toolSpecs);

  // tool_callsなし → 最終回答
  if (turn.toolCalls.length === 0) {
    answer = turn.content;
    break;
  }

  // tool_callsあり → ツールを実行してループ継続
  for (const tc of turn.toolCalls) {
    const result = await runToolCall(tc);
    // 検索結果を累積し、後で出典生成に使用
    if (tc.name === "search_knowledge_base" && result.ok) {
      sources.push(...result.data);
    }
    working.push({ role: "tool", toolCallId: tc.id, content: JSON.stringify(result.data) });
  }
}

🔧 LLMツール一覧

🔑
LLMはOpenAI Function Calling仕様に従い、呼び出すツールを自動的に決定します。ツールはagent/src/tools/registry.tsで定義され、すべてのツールはkbClientを通じてDjango REST APIを呼び出します。
search_knowledge_base
機能：クエリワードに基づきベクトル類似度検索を行い、最も関連性の高いChunkリストを返す
パラメータ：query（文字列）、category（省略可、ドキュメント分類）
呼び出し：GET /api/search/?q={query}&category={category}
後処理：返却結果を距離閾値SEARCH_MAX_DISTANCEでフィルタリング
list_documents
機能：ナレッジベース内の全ドキュメントリストを取得
パラメータ：なし
呼び出し：GET /api/documents/
用途：ユーザーが「どんな文書がありますか」と聞いた際に呼び出される
get_document_detail
機能：特定ドキュメントの詳細情報を取得
パラメータ：document_id（数値）
呼び出し：GET /api/documents/{id}/
用途：通常list_documentsの後、ユーザーが特定ドキュメントの確認を要求した際に呼び出される

🗂️ セッション管理（SessionStore）

💡
なぜデータベースではなくメモリにセッションを保存するのか？
このプロジェクトは開発/デモ用途のため、メモリ保存で十分です。複数ユーザーのセッションを永続化する場合はRedisまたはデータベース保存に変更が必要です。現在はサービス再起動後、すべての会話履歴が消去されます。
セッションID生成
crypto.randomUUID()
保存方式
メモリ（Map）
永続化
サービス再起動で消去
セッションファイル
agent/src/session.ts

フロントエンドは最初の対話後にsessionIdを受け取り、以降のリクエストに同じIDを含めることで会話履歴が継続されます。sessionIdを含めない場合、サーバーが自動的に新しいセッションを作成します。
