開発環境の構築
Docker Compose + uv + Bun + Make
Info-Agentの開発環境をゼロから構築する手順です。PostgreSQL（pgvector）コンテナの起動、Python/Node依存関係のインストール、環境変数の設定、すべてのMakefileコマンドについて説明します。
Docker Desktop uv Bun Make OpenAI API Key 必須

📋 前提条件

💡
なぜ pip ではなく uv を使うのか？
uvはRustで実装されたPythonパッケージ管理ツールで、依存関係の解決速度がpipより10〜100倍高速で、仮想環境の管理も内蔵しています。本プロジェクトではuvを使って開発環境の一貫性を確保しています。

Docker Desktop
Pythonパッケージ管理
uv
Node.js実行環境
Bun
OpenAI API
API Key 必須

⚙️ インストール手順

1
リポジトリのクローン
Shell 
git clone https://github.com/haya790315/Info-Agent.git
cd Info-Agent
2
Python依存関係のインストール
Shell 
make install
# .venv/ を作成し requirements.txt の全パッケージをインストール
# 初回実行時は sentence-transformers モデル（約90MB）をダウンロード
⚠️
初回インストール時、sentence-transformers が paraphrase-multilingual-MiniLM-L12-v2 モデル（約90MB）を自動ダウンロードします。安定したネットワーク接続を確認してください。
3
Node.js依存関係のインストール（Bun）
Shell 
make agent-install
# cd agent && bun install と同等
# agent/node_modules/ に Hono、OpenAI SDK などをインストール
4
環境変数の設定
Shell 
cp agent/.env.example agent/.env
# agent/.env を編集して必要な API Key を入力

詳細な環境変数の説明は下の 環境変数の設定 セクションをご参照ください。

5
データベースの起動とマイグレーション実行
Shell 
# PostgreSQLコンテナ起動 + pgvector拡張機能の有効化
make db-start

# Djangoデータベース構造の作成（Document + Chunkテーブル）
make db-migrate
✅
db-start コマンドはPostgreSQLの準備が完了するまで待機した後、CREATE EXTENSION IF NOT EXISTS vector を自動実行します。pgvectorを手動で有効化する必要はありません。
6
全サービスの起動
Shell 
# Django (:8080) + Bun Agent (:3000) を同時起動
make dev-all
# Ctrl+C で両サービスを同時停止できます
AI対話インターフェース
http://localhost:3000
ナレッジベース Web UI
http://localhost:8080

🔑 環境変数の設定（agent/.env）

⚠️
OPENAI_API_KEY が唯一の必須項目です。その他の変数はすべて合理的なデフォルト値を持っており、初回起動にはAPI Keyの設定だけで十分です。
dotenv 
# ============ 必須 ============
OPENAI_API_KEY=sk-...            # OpenAI APIキー（必須）

# ============ 任意（デフォルト値あり）============
OPENAI_MODEL=gpt-5.4-mini       # 使用するLLMモデル
OPENAI_BASE_URL=                    # カスタムAPIエンドポイント（空=OpenAI公式）
KB_API_BASE_URL=http://localhost:8080  # DjangoナレッジベースAPIのアドレス
SEARCH_MAX_DISTANCE=1.0           # ベクトル検索距離の閾値（0=完全一致、2=反対）
変数名必須デフォルト値説明
OPENAI_API_KEY必須—OpenAI APIキー、Agent起動時に検証
OPENAI_MODEL任意gpt-5.4-miniLLM推論モデル名
OPENAI_BASE_URL任意空（公式API）OpenAI互換エンドポイント（ローカルLLMなど）
KB_API_BASE_URL推奨http://localhost:8080DjangoナレッジベースREST APIのアドレス
SEARCH_MAX_DISTANCE任意1.0コサイン距離の上限（小さいほど厳格）
💡
SEARCH_MAX_DISTANCE の調整目安：
デフォルトの 1.0 はすべての検索結果を受け入れます（コサイン距離 0〜2）。Agentが関連性の低い内容を返す場合は 0.5 に下げて検索を厳格化してみてください。検索結果が少なすぎる場合は 1.2 に上げると改善されます。

🔧 よくある問題と解決方法

🐳
make db-start が失敗する：コンテナが起動しない
Docker Desktopが起動・実行中であることを確認してください。docker info でステータスを確認できます。
🗄️
make db-migrate で接続エラー
先に make db-start を実行してPostgreSQLコンテナを起動してからマイグレーションを実行してください。
🤖
Agentが「API key not found」を返す
agent/.env に OPENAI_API_KEY が正しく設定されていること、キーに余分なスペースがないことを確認してください。
📄
PDFアップロード後に検索結果が出ない
PDFの処理には数秒かかります。ドキュメント一覧で「処理完了」のステータスになってから検索してください。画像型（スキャン）PDFはテキスト抽出できないため「処理失敗」と表示されます。
🔄
埋め込みモデル変更後に検索結果が異常になる
sentence-transformersのモデルを変更すると、既存の384次元ベクトルと新モデルの出力が互換しなくなります。make reembed を実行して全ChunkのベクトルをReGenerate計算してください。
