📦 サービスモジュール

🤖
AI Agentサービス
TypeScript / Bun / Hono — :3000
Hono Webサービス + AgentLoop。OpenAI Function Callingによってツールループを駆動し、マルチターン会話セッションを管理し、出典ソースを自動付与します。
AgentLoop Function Calling Session Hono
📚
ナレッジベースサービス
Python / Django — :8080
PDFアップロード、テキスト抽出（pymupdf）、ベクトル埋め込み（MiniLM）、pgvectorコサイン類似度検索を提供。REST APIおよびHTMX Web UIを提供します。
Django REST pgvector MiniLM HTMX
🛠️
環境構築
Docker Compose + Make
Docker ComposeでPostgreSQLを管理し、uvでPython依存関係、BunでNode.js依存関係を管理します。Makeコマンドで全サービスを一括起動できます。
Docker uv Bun Makefile
📖
使い方
Web UI + REST API
AI対話インターフェース（:3000）、PDFアップロード管理（:8080）、セマンティック検索、ドキュメントステータス追跡、Django管理画面の操作方法を説明します。
Chat UI PDF Upload Semantic Search Admin

⚡ クイックスタート

5つのコマンドで初回セットアップ完了：

Shell 
# 1. 依存関係のインストール
make install && make agent-install

# 2. OpenAI API Keyを設定
cp agent/.env.example agent/.env
# agent/.env を編集して OPENAI_API_KEY を設定

# 3. データベース起動とマイグレーション実行
make db-start && make db-migrate

# 4. 全サービスを起動
make dev-all

# 5. ブラウザで確認
# AI対話：http://localhost:3000
# ナレッジベース：http://localhost:8080
⚠️ 注意：OpenAI API Keyを事前に取得し、agent/.env に設定していないと、Agentサービスが起動しません。
